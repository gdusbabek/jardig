Jardig Help

  1. The Help button in the upper right produces this message.
  2. Enter the name of the class you are looking for in the Class Name: field.
  3. Check your options:
    1. Recurse Directories tells the digger to exhaustively search subdirectories.
    2. Recurse Archives tells the digger to dig into archives nested inside other archives (.ear and .war for example).
    3. Summarize at end is useful if you have Log INFO turned on. It prints a succint summary of the results after scanning is finished.
    4. Log INFO turns on rich messages. You will be notified every time a directory is entered or a file is scanned.
    5. Log ERROR logs error conditions.
    6. Log EXCEPTION logs exceptions.
    7. Log FIND logs class name matches. You must have this or Summarize at end turned on in order to find out the results of your scan.
    8. Log NEST enables additional output when recursively scanning archives within other archives.
  4. Drag the directories you wish to scan into the Directories to scan list. If you wish to remove a directory, select it and press delete (or backspace).
  5. Click the Go! button.
  6. You should see output (as much as you opted for) in the Scan log pane on the bottom. To clear the output pane, triple-click it.

Source, javadocs and this help document are included in the jar that contains the application.
